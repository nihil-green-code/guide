# 쉼표, 그룹 연산자
이 문서에서는 쉼표 연산자와 그룹 연산자에 대해서 알아본다. 사실 별 거 없는 부분인데 그래도 다루고 넘어가도록 한다.

## 그룹 연산자
우리는 나중에 연산자들의 우선 순위에 대해서 다루게 될 건데 우선 순위 중에서 가장 높은 게 그룹 연산자이다.

```js
console.log(5 * 2 + 2);
// 10 + 2 = 12;
```

**보통**의 경우 **곱셈이 먼저 연산**된다. 하지만 **괄호를 쳐주면** 괄호 **안에 있는 내용물을 먼저 연산**한다는 것은 알고 있을 것이다. **그 괄호가 그룹 연산자**이다. 그룹 연산자는 나중에도 정말 유용하다. 연산자의 우선 순위를 하나 하나 외우는 바보같은 짓은 할 필요가 없고 그룹 연산자를 이용해서 처리해주는 게 훨씬 합리적일 것이다.

```js
console.log(5 * (2 + 2));
// 5 * 4 = 20;
```

## 쉼표 연산자
다소 생소하게 다가올 수 있는 부분이다. 아래의 코드를 보면서 생각해보자.

```js
let num1, num2, num3;
num1 = (num2 = 2, num3 = 3);

console.log(num1);
// 3
console.log(num2);
// 2
console.log(num3);
// 3
```

2번 라인에 보면 `num1`에 `(num2 = 2, num3 = 3)`를 넣는 것을 볼 수 있다. 괄호 안에서 쉼표 연산자가 사용되었다. 쉼표 연산자는 **각각의 피연산자들을 왼쪽부터 차례대로 평가**한 후에, **가장 마지막 피연산자를 리턴**하는 특징이 있다.

즉 num2 = 2는 평가가 되어 num2에는 2가 할당되고, num3 = 3이 평가가 되서 3이 할당된다. 그리고나서 num3의 값이 num1에서 들어간 것이다. 쉼표 연산자는 여러번 사용해보면 감이 온다. 대충 연습해보자.
